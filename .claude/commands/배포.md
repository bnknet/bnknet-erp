현재 브랜치의 변경사항을 이 레포의 배포 원칙대로 운영(main)에 반영한다.

반드시 순서대로:
1. `npx tsc --noEmit`와 `npm run build`가 통과하는지 확인. 실패하면 멈추고 그대로 보고한다.
2. 변경을 명확한 한국어 커밋 메시지로 커밋한다(코드/PR/커밋에 AI 모델명 노출 금지). `git push -u origin <현재브랜치>`.
3. `mcp__github__create_pull_request`로 base=main, head=현재브랜치 PR 생성.
4. `mcp__github__merge_pull_request`로 squash 머지.
5. 브랜치 재동기화: `git fetch origin main && git checkout -B <현재브랜치> origin/main` → `git push --force-with-lease`.
6. 머지·검증이 끝난 뒤에만 "완료"라고 보고한다. 실패는 실패라고 사실대로 보고한다.

$ARGUMENTS 가 있으면 커밋/PR 제목의 맥락으로 사용한다.
