ERP 데이터 양을 점검해 사용자에게 리포트한다. 목적: 데이터가 과도하게 쌓이거나 성능·백업에 문제될 조짐을 조기에 알림.

방법(직접 DB 접근 불가 → 사용자가 Supabase SQL Editor에서 실행하게 한다):
1. 아래 SQL을 복붙 가능한 형태로 제시한다.
2. 사용자가 준 결과를 해석한다. 행 수가 큰 테이블(특히 orders, approval_items, inventory_logs, order_change_logs)을 눈여겨본다.
3. 급증/비정상 신호가 보이면 원인과 대응(오래된 데이터 아카이브·인덱스·정리)을 제안한다. 문제 없으면 "여유 있음"이라고 안심시킨다.

제시할 SQL(테이블별 대략 행 수):

select relname as 테이블, n_live_tup as 대략_행수
from pg_stat_user_tables
order by n_live_tup desc;

백업 점검 안내도 함께:
- Supabase 대시보드 → Database → Backups 에서 자동 백업(일일/PITR) 활성 여부를 확인하도록 안내한다.
- 무료 요금제는 백업이 제한적일 수 있음 → 중요 데이터는 각 화면의 엑셀 내보내기로 정기 백업을 권한다.
- 참고 임계치(대략): 단일 테이블 수십만 행부터는 조회 속도·비용을 신경 써야 함. 그 전에 미리 알린다.
