## 1. 주문변환 도매 예상 마진 강조 — PR #80

## 2. 주문 삭제 시 재고 복구 + 매출 제외 — PR #81
- 문제: 기존 '완전 삭제'는 주문 행만 DELETE → 차감했던 재고가 복구 안 돼 재고 과소 계상
- **deleteOrdersWithRestore**: 삭제 전 `cancel_orders` RPC로 차감 재고를 원자적 복구 후 행 삭제
  (cancel_orders는 이미 취소된 건 건너뜀 → 이중복구 없음). 복구 실패 시 삭제 중단(정합성 보호)
- 조회·취소 탭 '완전 삭제' + **도매 주문 수정 모달에 '🗑️ 이 주문 삭제(재고 복구+매출 제외)' 버튼**(대표·실장)
- 매출은 주문 행이 사라지면서 자동 제외
- ※ 변경 이력(로그)은 감사기록이라 주문 삭제와 분리 유지 — 실제 삭제는 위 버튼 사용

## 3. 카드 결제 캘린더 선결제 카드 연동 오류 (데이터 교정)
- 증상: 6/30 조현상 국민카드 선결제인데 캘린더 상세엔 현대카드로 표시
- 진단: 코드 정상. 결재 작성 시 카드를 **현대카드로 잘못 선택**해 저장됨(선결제 문서 card_id=현대카드)
  - 캘린더·한도는 결재문서(approvals) 기준 실시간 계산이라 card_id를 그대로 반영한 것
- 교정: 해당 선결제(강웅구 정리, 3,664,500원, 6/30)의 card_id를 조현상 국민카드로 UPDATE
  - 검증 완료: card_name=국민카드 / holder=조현상 / card_company=국민
- ※ 조현상 카드들 card_type='조현상카드'는 홀더 그룹 방식(의도된 것) — 손대지 않음

## 4. 결재 결제카드 교정 기능 — PR #82
- 카드구매/선결제 결재에서 카드 잘못 선택 시, **결재 상세 → 💳 결제카드 옆 '카드 수정'** 버튼으로
  대표·실장이 직접 변경 (SQL 불필요)
- 카드 변경 시 **결제예정일을 새 카드 마감·결제일 기준으로 재계산**(등록 시와 동일 규칙:
  카드구매=발행일, 그 외=지출일 기준)
- card_id·payment_due_date만 PATCH → 결제 캘린더·잔여한도는 실시간 계산이라 자동 반영
- 변경 이력을 card_logs에 '카드변경'(이전→이후 카드·결제예정일)으로 기록
