# 슬랙 ERP 비서 봇 — 구축 가이드 (MVP)

> 대표·실장이 슬랙 DM으로 ERP 데이터를 자연어로 물어보면, Claude가 읽기 전용으로
> 조회·추론해서 답하는 봇. 1단계(전체 권한, DM 전용)부터 시작.

## 구조 한눈에
- **두뇌(백엔드)**: ERP(Next.js/Vercel) 안에 API 하나 추가 — `/api/assistant`
  - 슬랙 요청 수신 → 서명검증 → 화이트리스트 확인 → Claude(도구: ERP 읽기전용 조회) → 답변 게시
- **입출구**: 슬랙 DM (지금) → 나중에 채널·텔레그램 확장
- **보안**: 권한은 프롬프트가 아니라 백엔드가 강제. 조회는 **읽기(GET) 전용**, rpc/쓰기 차단.
  화이트리스트(`SLACK_ALLOWED_USER_IDS`)에 있는 슬랙 계정만 사용 가능.

엔드포인트 URL: **`https://bnknet-erp.vercel.app/api/assistant`**

---

## 대표님이 하실 일 (계정·키 — 이것만 해주시면 됩니다)

### A. Anthropic API 키 발급
1. https://console.anthropic.com → 로그인 → **API keys** → **Create Key**
2. 결제수단 등록(사용한 만큼 과금). 키 복사: `sk-ant-...`

### B. 슬랙 앱 생성
1. https://api.slack.com/apps → **Create New App → From scratch**
   - 이름: `ERP 비서` / 워크스페이스: 우리 회사
2. **Basic Information → App Credentials → Signing Secret** 복사
3. **OAuth & Permissions → Bot Token Scopes** 에 추가:
   - `chat:write`, `im:history`, `im:read`, `users:read`
4. 상단 **Install to Workspace** → 설치 → **Bot User OAuth Token** 복사: `xoxb-...`
5. **App Home** → "Messages Tab" 켜고,
   *"Allow users to send messages from the messages tab"* 체크 (DM 받게)
6. **Event Subscriptions → Enable Events**
   - Request URL: `https://bnknet-erp.vercel.app/api/assistant` → **Verified** 확인
     (※ 아래 D의 환경변수·재배포가 끝난 뒤 입력해야 검증됩니다)
   - **Subscribe to bot events**: `message.im` 추가 → **Save Changes**
   - (스코프/이벤트 변경 후 앱 재설치 안내가 뜨면 재설치)

### C. 내 슬랙 User ID 확인 (대표·실장 둘 다)
- 슬랙 프로필 → **⋯ 더보기 → Copy member ID** → `U`로 시작하는 값

### D. Vercel 환경변수 등록 (Project → Settings → Environment Variables)
| 이름 | 값 |
|---|---|
| `ANTHROPIC_API_KEY` | `sk-ant-...` |
| `SLACK_SIGNING_SECRET` | (B-2) |
| `SLACK_BOT_TOKEN` | `xoxb-...` (B-4) |
| `SLACK_ALLOWED_USER_IDS` | 대표·실장 User ID를 쉼표로 (예: `U01ABC,U02DEF`) |

`NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY` 는 이미 있음.
→ 저장 후 **Redeploy**.

### E. 테스트
- 슬랙에서 `ERP 비서` 검색 → **DM 열기** → 예: "이번달 매출 요약해줘",
  "덴프스 SJ글로벌 재고 얼마야?", "오늘 미차감 주문 있어?"
- 화이트리스트에 없는 계정이 말 걸면 "권한 없음" 안내가 갑니다.

## 순서 팁
1. **먼저** ERP 배포(코드 반영) → 2. Vercel 환경변수 4개 등록 → Redeploy
3. 그 다음 슬랙 앱의 **Event Subscriptions Request URL** 입력(검증) → `message.im` 구독
4. 대표·실장 DM으로 테스트

## 보안 메모
- 봇은 우리 ERP의 **anon 키**로 읽으므로, 앱이 못 읽는 데이터는 봇도 못 읽음.
- 조회는 GET 전용 — 데이터 수정/삭제/함수호출(rpc) 차단.
- 화이트리스트 밖 계정은 아예 답을 받지 못함.
- (다음 단계) 감사 로그 전체 기록, 채널별 주제 격리, 역할별 도구 제한, 텔레그램 개인비서.
