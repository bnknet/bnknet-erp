## 3. 카드 한도 처리 방식 정리 (논의 — 개발 없음)
- 결론: 현재 로직 그대로가 정확. 이중복구 없음
  - **6/30 이전 구매** → 결제일에 그 카드로 **선결제(한도복구)** 올려 승인 → 한도 복구
  - **6/30 이후 구매** → **카드구매(매입)** → 차감, 결제예정일 지나면 자동 복구
  - 두 경로가 안 겹쳐서 이중복구 없음. 6/30 이전분 소진되면 이후 건만 남음
- 회사 프로세스: 카드매입=매입품의서(카드구매)로만, 선결제=선결제 탭, 카드값 나가는 날 지출결의서는 기록용(한도 무관)

## 4. 지출결의서 결제카드·구매처 란 제거 (PR #94)
- 지출결의서에 카드 연결로 한도가 잘못 차감되는 소지 원천 차단
- 결제카드/구매처 입력란은 카드구매(선결제 포함) 전용 → 지출결의서엔 미노출
- 저장 시 지출결의서는 card_id·purchase_vendor·payment_due_date 강제 null
- 역할 정리: 매입품의서(카드구매)=한도차감 / 선결제=한도복구 / 지출결의서=지출기록(한도 무관)

## 5. 매출현황 브랜드별 매출 사업자 필터 (PR #96)
