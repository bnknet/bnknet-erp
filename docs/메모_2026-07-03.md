## 2. 결재 승인메모 배지 + 카드 상세 UI + 담당자·카드 표기 (PR #93)
- **승인 메모 배지**: 승인 지시·요청사항 메모 있는 결재 건에 목록에서 📝 표시(데스크탑·모바일)
- **결제 상세 모달 잘림 수정**: 화면 100%에서도 안 잘리게(상단 정렬 + max-h 스크롤, 긴 품목은 내부 스크롤)
- **담당자·카드 표기**: 결제카드를 '담당자 · 카드명'(예: 조현상 · 국민카드)으로
  — 결재 문서 상세, 결제 캘린더 상세/기간조회/결제상세 모달

## 3. 카드 한도 처리 방식 정리 (논의 — 개발 없음)
- 결론: 현재 로직 그대로가 정확. 이중복구 없음
  - **6/30 이전 구매** → 결제일에 그 카드로 **선결제(한도복구)** 올려 승인 → 한도 복구
  - **6/30 이후 구매** → **카드구매(매입)** → 차감, 결제예정일 지나면 자동 복구
  - 두 경로가 안 겹쳐서 이중복구 없음. 6/30 이전분 소진되면 이후 건만 남음
- 회사 프로세스: 카드매입=매입품의서(카드구매)로만, 선결제=선결제 탭, 카드값 나가는 날 지출결의서는 기록용(한도 무관)

## 4. 지출결의서 결제카드·구매처 란 제거 (PR #94)
- 지출결의서에 카드 연결로 한도가 잘못 차감되는 소지 원천 차단
- 결제카드/구매처 입력란은 카드구매(선결제 포함) 전용 → 지출결의서엔 미노출
- 저장 시 지출결의서는 card_id·purchase_vendor·payment_due_date 강제 null
- 역할 정리: 매입품의서(카드구매)=한도차감 / 선결제=한도복구 / 지출결의서=지출기록(한도 무관)

## 5. 매출현황 브랜드별 매출 사업자 필터 (PR #96)
- 사업자 버튼 선택 시 브랜드별 매출도 그 사업자 현재분으로 필터
- 과거 브랜드실적(brand_sales)은 사업자 구분 없어 전체일 때만 합산(특정 사업자 선택 시 제외)
- 7월부터 직접 변환·등록하는 매출은 사업자별로 정확히 분리됨(과거는 데이터 한계 — 인지됨)

## 6. 대화·업무 지침 저장 (PR #95, #97)
- docs/대화_업무_지침.md: 다정한 존댓말 말투 + 업무 응답 형식 + 워크플로우 + 개발 원칙
- 이모지는 아주 가끔만(톤다운)

## 7. 결재 상신자 수정/상신취소 + 대표·실장 전용 이력 (PR #99)
- 상신자 본인이 결재중·반려 문서를 수정(재상신, 결재라인 초기화) 또는 상신 취소(status=canceled) 가능
- 변경 이력(수정재상신·상신취소·승인철회)을 approval_logs에 기록, 상세에서 대표·실장만 조회(🔒)
- SQL: db/approval_logs.sql

## 8. 실장 승인 철회 (PR #100)
- 실장이 승인 후 대표 최종결재 전이면 '승인 철회'로 되돌려 재결재 대기 → 반려/수정 가능
- 대표 최종결재 후에는 불가(안전). approval_logs에 '승인철회' 기록

## 9. 인사관리 접근 제한 + 연봉/급여정보 (PR #101, #103)
- 인사 관리 접근을 대표·실장(ceo/admin)만 (페이지 가드 + 사이드바 roles)
- 직원 연봉(salary) + 급여일(pay_day) + 급여통장(salary_bank) 필드 추가, 폼 입력·상세 표시
- 6명 값 반영. SQL: db/employee_salary.sql, db/employee_pay_info.sql

## 10. 매입품의서 결재일자 승인 게이트 (PR #102)
- 카드구매 매입품의서(비선결제)는 settle_date(결재일자) 이후에만 승인 가능
- 발의(구매일)와 카드사 확정 매입금액(청구할인 등, 영업일 지연) 시점 차이 대응
  → 결재일 전까지 담당자가 금액 확인·수정. 승인/빠른승인 게이트, 반려는 상시 가능
- 선결제·지출결의서·휴가는 영향 없음

## 11. 결재 목록 상세 조회 필터 + 표시 건수 (PR #104)
