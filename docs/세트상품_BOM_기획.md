# 세트상품(BOM) 구성 자동 차감 기획

> 배경: 픽프롬 슬로우에이징 리겐 올인원 "본품+리필"처럼 1개 주문이 여러 구성품(본품·리필)
> 재고를 각각 차감해야 하는 세트상품 지원. 공동구매·옵션으로 세트 종류가 늘어날 예정.
> 작성: 2026-07-01 / 상태: **설계 확정 대기**

## 목표
- 세트상품 주문 시 구성품 재고를 **주문수량 × 구성수량**으로 각각 자동 차감
- 취소/취소해제 시 구성품 재고를 각각 정확히 복구 (정합성 유지)
- 매출은 세트 판매금액 그대로, 원가는 구성품 원가 합산
- 일반(1:1) 상품 로직은 **변경 없이** 그대로 (세트만 새 경로 → 기존 안정성 보존)

## 데이터 모델

### product_bom (세트 구성표)
| 컬럼 | 설명 |
|---|---|
| set_name | 세트 대표상품명 (matchProduct 결과와 동일) |
| component_name | 구성품 재고 상품명 (inventory.product_name과 일치) |
| component_qty | 세트 1개당 구성품 수량 (예: 본품 1, 리필 1) |
| company | (선택) 사업자별 구성 다르면 지정, 없으면 공통 |

- 구성품 재고 매칭: `component_name | order.company` → inventory_id (기존 매칭 규칙과 동일, 폴백 포함)

### order_deductions (주문별 다중 차감 이력)
| 컬럼 | 설명 |
|---|---|
| order_id (bigint) | 주문 |
| inventory_id (uuid) | 차감된 재고 |
| product_name | 구성품명(로그용) |
| qty | 차감 수량 |

- 세트 주문은 여기에 여러 줄. 취소 시 이 표를 읽어 각 구성품 복구.
- 일반 상품은 기존 `orders.deducted_inventory_id/deducted_qty` 그대로 사용(변경 없음).

## 처리 흐름
1. 변환/재출고 시: `matchProduct(수집명)` → 대표명. 이 대표명이 `product_bom.set_name`에 있으면 **세트**.
2. 세트면 구성품 목록을 조회 → 구성품별 `inventory_id` 해석 → 주문 1건에 대해 **여러 차감** 지시 생성.
3. RPC `ship_orders`(확장): 세트 주문은 구성품마다 재고 차감 + `order_deductions` 기록 + `inventory_logs` 기록, 주문은 `stock_deducted=true` 1회 마킹.
4. `cancel_orders`/`uncancel_orders`(확장): `order_deductions`에 이력 있으면 그걸로 각 구성품 복구/재차감, 없으면 기존 단일 칸 로직(하위호환).

## 원가·매출 규칙 (확정 필요)
- 매출: 세트 판매금액 그대로 (구성품으로 안 나눔). **← 확정 제안**
- 원가: `∑(구성품 원가 × 구성수량 × 주문수량)`. 매출현황 원가/공헌이익에 반영. **← 확정 제안**
- 세트 대표상품 자체는 재고를 두지 않음(구성품 재고만 실재). 매출집계는 세트명 기준.

## 재고 부족 시 정책 (확정 필요)
- 구성품 중 일부만 재고가 있을 때:
  - (A) 있는 것만 차감하고 부족분은 마이너스 허용 + 경고 (기존 마이너스 정책과 동일) **← 제안**
  - (B) 하나라도 부족하면 전체 보류(미차감) + 알림

## 화면
- 상품마스터에 "세트 구성 관리" (세트명 → 구성품·수량 등록/수정). 초기엔 SQL 시드로 시작 가능.

## 개발 순서
1. `product_bom` + `order_deductions` 테이블 (SQL)
2. `ship_orders`/`cancel_orders`/`uncancel_orders` 세트 경로 확장 (하위호환 유지)
3. 프론트: 세트면 구성품 다중 차감 지시 생성 (변환·재출고)
4. 세트 구성 관리 UI
5. 매출현황 원가 = 세트 구성품 합산 반영

## 시드에 필요한 정보 (사용자 제공)
- 세트상품 옵션 목록(공동구매 포함) 각각의 정확한 대표상품명
- 각 세트의 구성품과 세트당 수량 (예: 본품 1 + 리필 1)
- 구성품이 재고(상품마스터)에 등록된 정확한 상품명 + 사업자
