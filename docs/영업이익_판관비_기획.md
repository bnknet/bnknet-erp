# 판관비 반영 "진짜 영업이익" 기획 메모 (7월 이후 진행)

> 트리거: 사용자가 "판관비 영업이익 만들자"라고 하면 이 문서 열어서 진행.
> 작성: 2026-07-01 / 상태: **보류(기획만)**

## 배경
현재 매출현황의 지표는 **공헌이익**이다.
```
공헌이익  = (상품 + 고객배송비 − 몰수수료 − 원가 − 실운임) ÷ 1.1   ← 변동비만 차감
공헌이익률 = 공헌이익 ÷ 매출
```
여기엔 **고정 판관비(인건비·임대료·광고비·PG수수료 등)가 빠져 있어** 재무제표상 영업이익과 다르다.
→ 고정 판관비를 월별로 입력받아 한 단계 더 차감하면 **진짜 영업이익 / 영업이익률**을 낼 수 있다.

## 목표
1. 월별 고정 판관비를 입력·관리하는 화면
2. `영업이익 = 공헌이익 합계 − 월 고정판관비` 계산
3. **핵심 인사(경영진)만 열람** — 권한 제한 (일반 직원은 공헌이익까지만)

## 필요 데이터 (판관비 항목 예시)
| 항목 | 성격 | 부가세 |
|---|---|---|
| 인건비(급여·상여) | 고정 | 면세(급여) |
| 4대보험·퇴직충당 | 고정 | - |
| 임대료·관리비 | 고정 | 과세 |
| 광고선전비(마케팅) | 준변동 | 과세 |
| PG/카드 결제수수료 | 변동 | 과세 |
| 포장재·부자재 | 변동 | 과세 |
| 창고·물류 고정비 | 고정 | 과세 |
| 세무·회계·수수료 | 고정 | 과세 |
| 기타 잡비 | - | - |

※ 부가세 처리: 판관비도 과세분은 ÷1.1(매입세액 공제), 급여 등 면세분은 그대로.

## 계산 구조
```
[사업자별]
  월 공헌이익 합계 (매출현황에서 자동 집계)
  − 월 고정판관비 (직접 배부 + 공통비 배분)
  = 월 영업이익
  영업이익률 = 영업이익 ÷ 매출
```
- **공통비 배분 기준**: 여러 사업자 공유 비용(본사 임대료 등)은 매출비례 or 인원비례로 배분. 배분 기준을 설정값으로.
- **직접비**: 특정 사업자에만 귀속되는 비용은 그 사업자에 직접 배부.

## DB 설계(안)
```
opex (판관비) 테이블
  id, company, year, month, category, amount, is_taxable(과세여부),
  alloc_type('direct'|'shared'), memo, created_by, created_at
opex_alloc_rule (공통비 배분 규칙)
  company, basis('revenue'|'headcount'|'fixed_ratio'), ratio
```

## 화면(안)
1. **판관비 입력** (월 선택 → 사업자별·항목별 금액 입력, 카드결제/전월복사 기능)
2. **손익 요약(경영진 전용)**: 매출 → 공헌이익 → (−판관비) → 영업이익 폭포수/표
3. 매출현황 화면엔 노출 안 함(권한 분리)

## 권한
- 현재 권한은 client-side(sessionStorage)라 완전하지 않음 → **7월 이후 보안 마이그레이션(docs/보안_현황및계획.md)과 함께** 서버 권한(RLS/role)으로 잠금.
- 임시로는 employees.role in ('대표','경영지원') 등 특정 롤만 메뉴 노출.

## 개발 순서(제안)
1. opex 테이블 + 입력 화면 (MVP: 사업자·월·항목·금액)
2. 월 공헌이익 자동 합산과 연결 → 영업이익 산출
3. 경영진 전용 손익 요약 뷰
4. 공통비 배분 규칙(선택 기능)
5. 보안 마이그레이션으로 열람 권한 서버단 잠금

## 열린 질문 (진행 시 확인)
- 판관비를 **월 단위**로만 볼지, 분기/연 롤업도 필요한지
- 광고비를 사업자별로 나눌 수 있는지(믹스 광고면 배분 필요)
- 급여 등 민감정보를 ERP에 넣을지 vs 총액만 넣을지
