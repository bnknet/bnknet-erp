## 1. 수량표기 확인 안전망 (A+B) — PR #76

기존 치환 로직은 **변경 없이 유지**(검증 완료, 현재 카탈로그 위험 0건)하고 **감지·검수 계층만 추가**.

  미인식 수량단위 행을 노란 배경 + ⚠️ 사유로 표시 → 송장 출력 전 눈으로 검수
- **B. 자동 감지 알림**: 변환 완료 시 "수량 확인 필요 N건" 경고 + DB 저장 시
  `ship_alerts(kind=qty_check)` 기록 → 대시보드/주문탭 안전망 배너 노출
- 대시보드 `SHIP_ALERT_LABEL`에 `qty_check` 라벨 추가

## 3. 상품 매칭 셀프 관리 — PR #78 ⭐
매칭데이터 파일을 담당자→대표→개발자로 주고받던 병목 제거.
신규 상품이 들어오면 담당자가 ERP에서 직접 매칭 추가 → 새로고침하면 즉시 반영·알림 해소.
- **`product_matches` 테이블** 신설 + 변경 로그(`product_match_logs`)
- **matchProduct 병합 로직**: 검증된 하드코딩 820개를 base로 유지하고 DB 매칭을 덧씌워 병합
  (DB 우선 → 담당자 교정 허용). 조회 실패 시 하드코딩만으로 정상 동작 → **기존 매칭 절대 안 깨짐**
- **loadDbMatches()**: 주문변환·매출·대시보드·재고 집계 직전에 최신 매칭 로드
- **/product-matches 관리 화면**: 추가/수정/삭제 + 현재 매칭 미리보기 + 변경 이력
  - 권한: 대표·실장·영업·재고담당(세트 구성과 동일), 로그 삭제는 대표·실장
  - 매칭 삭제는 등록 가능자 모두(담당자가 본인 실수 정정)
- **미매칭 알림에 `🔗 매칭 추가` 바로가기**(주문변환 미매칭 박스 + 대시보드 알림)
- 사이드바 '재고·상품' 그룹에 '상품 매칭' 메뉴 추가

### 재고·매출 연동 확인 (중요)
- 재고 차감과 매출 집계가 **동일한 `matchProduct()` 하나**를 기준으로 대표명을 뽑음
  → 매칭 추가 시 재고·매출이 **같은 기준으로 동시에** 움직임 (따로 놀 수 없음)
- **규칙**: 매칭의 대표상품명 = 재고에 등록된 상품명과 글자까지 동일해야 재고 자동차감.
  화면에서 재고 목록 자동완성 + 재고에 없으면 ⚠️재고없음 경고. 실수해도 매출만 잡히고
  재고 미차감+알림(오출고 아님)
- **시점**: 매출·원가는 새로고침하면 과거 주문까지 소급 반영 / 재고는 앞으로 변환분 자동,
  과거 미차감분은 '재출고(재고 미차감분)' 버튼으로 소급 차감
- **송장 수량 치환 로직은 미변경** — 매칭은 '어떤 상품(이름)'만 결정, 수량은 기존 로직 그대로
