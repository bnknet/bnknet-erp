# Slack 알림 — 쉬운 설명서 (2026-06-30)

> ERP에서 무슨 일이 생기면 Slack으로 자동 알림이 오게 만든 기능. 나중에 누구한테 설명할 때 이 문서 보면 됨.

## 1. 한 줄 요약

**ERP(Supabase DB)에서 특정 사건이 생기면 → Slack 채널(`#erp-알림`)로 메시지가 자동으로 날아온다.**
한 방향(ERP → Slack)이라 "스피커"처럼 알림만 보냄. Slack에서 명령을 받진 못함.

## 2. 어떻게 작동하나 (비유)

```
[ERP에서 사건 발생]
   예) 주문 변환 중 재고 매칭 실패 / 결재 상신 / 재고 부족
        │
        ▼
[Supabase DB의 "트리거"가 감지]   ← 데이터가 들어오면 자동 실행되는 감시병
        │
        ▼
[pg_net이 Slack으로 메시지 전송]  ← DB가 인터넷으로 메시지 쏘는 도구
        │
        ▼
[Slack #erp-알림 채널에 알림 도착] 📲
```

- **트리거(trigger)**: "이 테이블에 새 데이터가 들어오면 이걸 해라"라고 DB에 걸어둔 자동 규칙.
- **pg_net**: DB가 외부 인터넷 주소로 메시지를 보낼 수 있게 해주는 Supabase 확장 기능.
- **Webhook URL**: Slack이 발급한 "이 주소로 메시지 보내면 이 채널에 올려줄게" 하는 비밀 주소.

## 3. 만든 순서 (재현 방법)

1. **Slack 앱 생성** — api.slack.com/apps → Create New App → From scratch (앱 이름 'ERP알림', 워크스페이스 Bang)
2. **봇 권한 추가** — OAuth & Permissions → Bot Token Scopes에 `incoming-webhook` 추가
   - ⚠️ "설치할 봇 사용자 없음" 에러 나면 → App Home에서 Display Name 저장(봇 사용자 생성) 후 설치
3. **워크스페이스 설치** — Install to Workspace → 허용
4. **Webhook 발급** — Incoming Webhooks → Add New Webhook to Workspace → 채널 `#erp-알림` 선택 → URL 복사
5. **DB에 트리거 SQL 실행** — Supabase SQL Editor에서 함수+트리거 생성 (URL 입력)

## 4. 현재 알림 오는 사건들

| 사건 | 언제 | 메시지 |
|---|---|---|
| 🟠 자동출고 실패/미매칭/재고부족 | 주문 변환 시 재고 매칭 안 됨 | ⚠️ BNKnet ERP 자동출고 알림 |
| 📝 결재 상신 | 결재 문서 새로 올라옴 | 새 결재 상신 (문서·사업자·상신자·금액) |
| 📦 재고 부족 경고 | 재고가 기준(5개) 이하로 떨어짐 | 재고 부족 경고 (상품·남은 수량) |
| 🚨 매출 이상 | 매일 20시, 주문은 있는데 매출 0원 | 매출 이상 징후 |

## 5. 관련 파일 / 설정

- `db/ship_alerts.sql` — 알림 기록 테이블
- `db/ship_alerts_slack.sql` — 자동출고 알림 → Slack (초기)
- `db/slack_alerts_expand.sql` — 결재·재고부족·매출이상 알림 + 공용 함수 `slack_notify()`
  - **Webhook URL은 `slack_notify()` 함수 한 곳에서만 관리** (바꿀 때 여기만 수정)
- Slack 앱: 'ERP알림' (App ID `A0BE06VJA13`, 워크스페이스 Bang, 채널 `#erp-알림`)

## 6. 중요 — 자주 묻는 것

- **Q. Slack 채널에 "이거 고쳐줘" 하면 되나?**
  A. ❌ 안 됨. 알림 전용(한 방향)이라 명령 못 받음. **코드 수정·기능 요청은 웹 Claude Code(개발 세션)로** 해야 함.
- **Q. 알림이 안 와요.**
  A. `select id, status_code, error_msg from net._http_response order by created desc limit 5;` 로 확인.
     `200`=정상, `403/404`=Webhook URL 무효(재발급 필요), 빈 결과=트리거/ pg_net 문제.
- **Q. Webhook URL이 노출됐어요.**
  A. Slack Incoming Webhooks에서 옛 URL 삭제 → 새로 발급 → `slack_notify()` 함수의 v_url만 교체.

## 7. 앞으로 확장 가능 (같은 방식)

- 카드 결제일 임박 알림, 연차 신청 알림, 일일 매출 요약 등 → `slack_notify('메시지')` 호출만 추가하면 됨.
