# Slack AI 분석 비서봇 — 구상 (나중에 구현)

> 2026-06-30 도출. 7/1 가동 안정화 후 단계적으로 진행. 지금은 구상만 기록.

## 1. 방향 (확정된 결정)

- **ERP = 입력·관리** (주문·재고·결재·매출 등록/조회 화면)
- **Slack 비서봇 = 분석·검색·추천** (물어보면 ERP 데이터로 답)
- 원래 ERP 안에 AI 서칭을 넣으려 했으나 → **ERP/Slack 분리**로 결정.
  분석·질문성 요청은 **Slack을 통해** 하는 게 맞다고 결론.

## 2. 목표 사용 예시 (대표님/실무자가 Slack에 입력)

> "3~6월까지 영업이익률 TOP3 뽑아주고, 원가는 얼마였고, 월별 출고수량 몇 개였는지 알려줘.
>  이걸 바탕으로 3개월치 예상재고는 몇 개 가져가야 하는지 추천해줘."

→ 자동으로 표 + 추천이 Slack에 나오는 것이 목표.

## 3. 작동 원리 (이 예시가 도는 흐름)

```
Slack 질문 💬
   │
   ▼
백엔드(중간 처리자)
   │
   ├─ 1) Claude API: 질문 해석 ("무슨 데이터가 필요한가?")
   │
   ├─ 2) Supabase: 실제 숫자 조회·계산
   │       · 3~6월 주문/재고 → 영업이익률 TOP3 정렬
   │       · 상품별 원가, 월별 출고수량 집계
   │
   ├─ 3) Claude API: 그 숫자로 추천·문장 작성
   │       · 예상재고 = 월평균 출고 × 3개월 + 안전재고
   │
   ▼
Slack 답변 💬 (표 + 추천)
```

## 4. ⭐ 핵심 원칙 — 역할 분담

| 구분 | 담당 | 이유 |
|---|---|---|
| 숫자·계산 (영업이익률, 원가, 출고량) | **Supabase(DB)** | 정확성. AI 암산은 틀릴 수 있음 |
| 추천·예측 (예상재고, 조언) | **Claude(AI)** | 판단·요약은 AI가 잘함. 단 "참고용 예측" |
| 질문 이해 / 답변 문장화 | **Claude(AI)** | 통역사 역할 |

→ **"사실은 DB, 판단은 AI"** 가 철칙. 추천 수치는 "예측/조언"이며 최종 결정은 사람이.

## 5. 필요한 것

| 항목 | 설명 | 비용 |
|---|---|---|
| 백엔드 서버 | Supabase Edge Function 또는 Vercel 함수 | 거의 무료~소액 |
| Slack 앱 (양방향) | Events API + 봇 토큰 (현재 Webhook은 알림 전용·한 방향) | 무료 |
| Claude API 키 | 질문 이해 + 답변 작성 | **질문당 토큰 과금** |
| 연결 코드 | Slack ↔ Claude ↔ Supabase | 개발 필요 |

## 6. 단계별 추천 진행안

1. **(현재) 알림 전용** — 자동출고·결재·재고부족·매출이상 Slack 알림 ✅ 완료
2. **1단계: 명령형 봇** — `/매출 5월`, `/재고 콜라겐` 같은 고정 명령 → 빠른 조회 (토큰 적음, 만들기 쉬움)
3. **2단계: 자연어 분석봇** — 위 예시처럼 자유 질문 → 분석·추천 (토큰 더 듦, 가장 강력)
4. **(확장) 비서봇/요약봇** — 일일 매출 요약 자동 발송, 결재 리마인드 등

## 7. 비용·주의

- Claude API는 **질문 1건당 토큰 과금** → 자주 쓰는 핵심 질문부터 시작이 가성비 좋음.
- 데이터 접근 권한 주의: 봇이 매출·원가 등 민감정보를 다루므로 **특정 채널/사용자로 제한** 권장.
- 추천 결과는 **참고용**. 발주 등 최종 결정은 사람이.
