## 1. 대화 기록 vs 작업물(코드) — 헷갈리지 말 것

- **작업물(코드)** = GitHub(git 커밋)에 저장 → **안전. 안 사라짐.**
  - 확인 결과: 로컬 = GitHub 완전 일치(커밋 `d388efb`), **총 50개 커밋 전부 보존**.
  - 만든 모듈 폴더 전부 그대로: inventory / sales / attendance / approval / partners / cards / products / hr 등.
- **대화 기록(채팅 로그)** = 세션·계정에 묶여 보관 → 환경 바뀌면 안 보일 수 있음(고장 아님).
- 맥락 유지 안전장치 = **PROJECT_PLAN.md "6. 진행 기록"**. 대화가 끊겨도 이 파일만 읽으면 작업 이어짐.

## 2. CLI(커맨더) vs 웹 Claude Code

- 교육은 **터미널(CLI)** 로 진행. 웹과는 다른 세션.
- **연습은 ERP 폴더 말고 별도 연습 폴더에서** 할 것 (예: `claude-practice`).
  - Claude Code는 "열린 폴더 안"에서만 동작 → 연습 폴더에서 하면 ERP 프로젝트 안 건드림.
  - 연습 폴더는 bnknet-erp 저장소와 연결하지 말고, **실수로라도 push 금지.**

## 3. 플러그인(plugin)이란?

- Claude Code에 기능 묶음(명령어·에이전트·자동화)을 추가 설치하는 것.
- `/plugin marketplace add <주소>` = 플러그인 "가게" 등록.
- 예시로 받은 `fivetaku/gptaku_plugins` = 한국 개발자가 만든 14개 플러그인 모음(연습용).
- ⚠️ **`/plugin`은 웹에선 안 됨. CLI(터미널)에서만 동작.**
- ⚠️ **제3자(외부인) 플러그인** → 그 사람 코드가 내 PC에서 돌 수 있음.
  **연습 폴더에서만 설치, 실제 ERP 작업 폴더엔 설치 금지.**

## 6. Supabase 데이터 안전성 + 백업 (중요)

### 안전성
- Supabase 데이터는 **클라우드(외부 서버)** 저장 → PC 끄기/폴더 변경/CLI·웹 전환과 **무관, 안 날아감.**
- **Supabase Pro(유료) 결제 중** → 매일 자동 백업 + 일시정지 위험 없음.
- 유일한 실질 위험 = **사람이 SQL로 실수로 삭제(DELETE/DROP)** 또는 RLS 잘못 풀기.

### 수동 백업 방법 (한 달 1회 규칙)
- **방법 A (쉬움, 정기용) ⭐**: 대시보드 → Table Editor → 테이블 선택 → Export → **Export as CSV**.
  주요 테이블 반복: orders / inventory / inventory_logs / employees / partners / cards / approvals.
  폴더명 `백업_YYYY-MM-DD`.
- **방법 B (전체 통백업, 완전 복구용)**: Project Settings → Database → Connection string 복사 →
  `pg_dump "연결문자열" -F c -f bnknet_erp_백업_YYYY-MM-DD.dump`
  (기술적 → 어려우면 7/1 직전에 함께 진행)

### 구글 드라이브에 백업 올릴 때 보안 (2겹)
- **1겹 — 파일 암호화(가장 강력)**: 백업 폴더를 **비번 압축(7-Zip, AES-256)** 후 그것만 업로드.
  비밀번호는 드라이브에 같이 올리지 말고 따로 보관(대표·실장만).
- **2겹 — 계정/폴더 권한**: 구글 계정 **2단계 인증(2FA)** 켜기 / 백업 폴더 공유 **"제한됨(대표·실장만)"** /
  **"링크 있는 사람 누구나" 절대 금지** / 가능하면 회사 Workspace 계정 사용.

### 백업 운영 규칙(요약)
- 매달 1회: CSV 백업 → 비번 압축 → 제한공유 드라이브 폴더 업로드.
- 7/1 직전 1회: 전체 .sql 백업으로 "정식 가동 기준점" 만들기.

## 10. 마진(영업이익) 계산법 — 확정 (2026-06-27, mall_fees SQL 적용 완료)

> 현재 매출 현황(/sales)에서 실시간으로 이렇게 계산됨. 모든 계산은 코드/DB가 정확히 수행.

**정석 공식 (2026-06-27 확정·가동 중) — 부가세 제외(공급가액) 기준:**
```
정산금액 = 매출(판매금액) − 몰수수료(매출 × 요율)
영업이익 = (정산금액 − 원가 − 운임) ÷ 1.1 + 고객배송비 × 0.967
영업이익률 = 영업이익 ÷ 판매금액 × 100   (판매금액 대비 — 타이트 관리용)
   ※ 고객배송비(택배비)는 부가세 대상 아님 → ÷1.1 미적용, 3.3% 세금만 차감(×0.967). (2026-06-27 사용자 확인)
```
- 핵심: 매출·원가·수수료·운임이 모두 부가세 포함이므로 **÷1.1로 부가세를 일괄 제거**해야 정확. (그냥 매출−원가−수수료는 과대계상)
- **원가 = 부가세 포함 개당원가 × 수량** (사용자 확인: 원가는 VAT 포함으로 입력됨 → ÷1.1 대상 맞음).
- **운임 = 건당 2,300원**, 과세운임(택배사 세금계산서 수취 → ÷1.1 대상). **합구매(같은 주문번호)는 1회만** 차감. 유·무료배송 무관 항상 차감.
- **고객배송비 = orders.delivery_fee** (고객이 낸 배송비, 0이면 무료배송). 주문당 1회.
- **0.033** = 고객배송비 부대 차감율(사용자 추후 조정 가능, mall 0이면 영향 없음).
- **몰수수료**: (사업자+판매몰)별 — `mall_fees` 테이블. 비앤/더블/SJ: 스마트스토어6/G마켓12/옥션12/11번가12/쿠팡10.56/토스12.10%, IX: 스마트스토어6/SSG35/Hmall15/롯데온25/자사몰Npay2.97/자사몰직접결제3.24%.
- **안전장치**: 원가 미입력/수수료율 미설정 시 경고 배너(0 처리 안 함). 영업이익은 원가 확인된 분만 반영.
- **검산**(스마트스토어 예시): 매출306,810·원가261,630·수수료6%·운임2,300 → 영업이익 **22,247원 / 7.25%** (수기 변경공식과 일치 ✅).

**아직 안 빠진 것 (2차):** 부가세 정밀(현재 ÷1.1 단순화)·ESM 정산 파일 매칭(주문별 실수수료 보정). 마진율 순매출(공급가액) 대비 보기는 추후 요청 시.

**요율/운임/0.033 변경 시:** mall_fees는 Supabase 표 수정, 운임·0.033은 `src/app/sales/SalesContent.tsx` 상수(UNIT_SHIPPING·CUST_SHIP_ADJ) 수정.

## 11. 수기(직접) 주문 등록 기능 — 제안 (2026-06-27, 다음에 구현)

> 사방넷 안 거치는 도매·직거래·기타 주문을 ERP에서 직접 등록하고 싶음.

**어디에**: 주문 모듈(`/orders`)에 탭 추가 `✍️ 직접 주문 등록`.

**입력 항목**:
- 사업자(드롭다운 필수), 판매몰/채널(+"도매"·"직거래" 추가), 상품(⭐재고에서 검색·선택), 수량, 금액, 날짜(기본 오늘·과거지정 가능), 수취인/거래처·배송비(선택). 한 주문에 여러 품목 줄 추가 가능.

1. **상품은 재고에서 골라 넣게**(직접 타이핑 ❌) → 상품명·원가·사업자 100% 일치 → 자동출고·원가매칭 안 깨짐.

3. 저장 시 변환 주문과 동일하게 orders 적재 → 매출집계 + 재고 자동차감 + 취소복구(3중 정합성) 재사용.
4. 권한: 재고·주문담당 + 대표·실장.
